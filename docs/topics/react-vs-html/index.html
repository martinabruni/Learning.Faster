Browser vs React build

Il browser capisce il risultato finale, non sempre il codice sorgente.

HTML, CSS e JavaScript classici sono già nel formato che il browser può leggere. React moderno, invece, spesso parte da JSX, moduli npm, import, TypeScript e asset: prima deve passare dentro una “macchina” che lo trasforma in file browser-ready.

file:///index.html oppure http://localhost:5173
Risultato nel browser

Due strade diverse per arrivare allo stesso punto

Alla fine il browser vuole sempre la stessa cosa: HTML, CSS e JavaScript standard. La differenza è quanto lavoro serve prima di arrivare lì.

A File HTML semplice

Qui il file che scrivi è già quasi il file finale. Il browser lo apre, legge i tag, carica CSS e JS, costruisce il DOM e disegna la pagina.

📄
index.html Contiene la struttura della pagina.
🎨
style.css Il browser interpreta direttamente le regole grafiche.
⚙️
script.js JavaScript standard eseguito direttamente dal motore JS del browser.
🌐
Browser Costruisce DOM, applica CSS, esegue JS e mostra la pagina.
Codice già leggibile dal browser apri e parte 
<!DOCTYPE html>
<html>
  <head>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1>Ciao</h1>
    <script src="script.js"></script>
  </body>
</html>

B Progetto React moderno

Qui tu scrivi codice più comodo per sviluppare, ma non sempre già digeribile dal browser. Serve un tool che trasformi tutto.

⚛️
Componenti React Scrivi componenti, stato, props, JSX e import.
📦
Dipendenze npm React, ReactDOM e altri pacchetti vengono presi da node_modules.
🏗️
Build tool Vite/Webpack/Parcel convertono JSX, risolvono import e preparano i file.
🚀
Bundle finale Il risultato diventa HTML, CSS e JavaScript standard.
🌐
Browser Ora sì: il browser riceve file che può caricare ed eseguire.
Codice sorgente React prima va trasformato 
import { createRoot } from "react-dom/client";
import "./style.css";

function App() {
  return <h1>Ciao</h1>;
}

createRoot(document.getElementById("root")).render(<App />);

Idea chiave: il browser non esegue “React come progetto”. Esegue il risultato della trasformazione: file statici finali, cioè HTML, CSS e JavaScript standard.

Tradotto in parole semplici

HTML classico

È come consegnare al browser un piatto già pronto. Deve solo servirlo: legge la pagina, applica lo stile ed esegue lo script.

React sorgente

È come consegnare ingredienti e ricetta. Sono utilissimi per chi cucina, ma il browser vuole il piatto già preparato.

Build finale

Il build tool cucina tutto: JSX, import npm, CSS, immagini e moduli diventano file statici caricabili dal browser.

In pratica: React non è “magico”. Alla fine anche lui deve diventare normale HTML, CSS e JavaScript.

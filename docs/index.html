Publishing

Come renderlo pubblico su GitHub Pages

Per questa repo non serve un workflow custom: GitHub raccomanda la pubblicazione da branch quando non esiste un build process dedicato.

1. Source

In repository settings, abilita GitHub Pages e seleziona Deploy from a branch.

2. Branch

Imposta main come branch e /docs come folder di pubblicazione.

3. URL

Il sito verrà pubblicato su https://martinabruni.github.io/Learning.Faster/.

Perché questa struttura

È la soluzione più semplice da mantenere oggi: niente branch dedicato, niente build step obbligatorio e nessun lock-in verso un generatore statico finché il contenuto resta HTML/CSS/JS puro.
